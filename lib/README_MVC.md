# MVC Pattern – File & Folder Structure

This project uses a **Model–View–Controller (MVC)** layout with Firebase (Firestore) as the backend.

## Folder structure

```
lib/
├── main.dart                 # Entry: Firebase init, runApp(App)
├── app.dart                  # MaterialApp, theme, home route
│
├── core/                     # Shared app config (no business logic)
│   ├── constants/
│   │   └── app_constants.dart
│   └── theme/
│       └── app_theme.dart
│
├── models/                   # Data models (domain only)
│   └── text_item_model.dart
│
├── views/                    # UI only – widgets and screens
│   └── home/
│       ├── home_view.dart
│       └── widgets/
│           ├── add_item_sheet.dart
│           └── item_tile.dart
│
├── controllers/              # Business logic, state, use services
│   └── home_controller.dart
│
├── services/                 # External I/O (Firebase, API)
│   └── firebase_service.dart
│
└── firebase_options.dart     # Generated by FlutterFire CLI
```

## Responsibilities

| Layer      | Role |
|-----------|------|
| **Model** | Data shape and serialization (e.g. `fromFirestore`, `toMap`). No UI, no Firebase calls. |
| **View**  | Builds UI, listens to a controller, calls controller methods on user actions. |
| **Controller** | Holds state, runs logic, uses **services** for Firebase/API, notifies listeners. |
| **Service** | All Firebase (or other backend) access. Single place for Firestore reads/writes. |

## Run the app

1. Ensure Firebase is set up (Firestore, and `flutterfire configure` if needed).
2. Create a Firestore collection named `items` (or change `AppConstants.collectionItems`).
3. Run: `flutter run`.

## Adding a new feature (MVC)

1. **Model** – Add a class in `lib/models/` with `fromFirestore` / `toMap` if using Firestore.
2. **Service** – Add methods in `lib/services/firebase_service.dart` (or a new service file) for Firestore/API.
3. **Controller** – Add a controller in `lib/controllers/` that uses the service and `notifyListeners()`.
4. **View** – Add a screen in `lib/views/` that uses the controller with `ListenableBuilder` and calls controller methods.
